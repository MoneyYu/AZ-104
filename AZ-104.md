【网络限制】Azure PaaS 服务默认都只提供公网的URL进行访问，若要进行vNET内部的访问：
*Service Endpoint
*Private Endpoint
*也可以通过限制特定的IP地址范围来限制客户端的访问

【身份验证】
* 匿名读取访问（在容器级别进行设定，也可以在存储账户级别进行强制开关）
* 访问密钥（在存储账户级别会有两个Key，也叫存储账户密钥、共享密钥。能获得整个存储账户中所有数据的访问权，注意一定在安全的应用中使用它，并保护好密钥，或者定期更新密钥）
* 共享访问签名（SAS，计算出一个签名（权限策略+访问密钥），组成token，授予我们所选择的访问权限）
* Azure AD 身份验证

【访问服务器 -> 存储账户】
IP（URL） -> Endpoint URL
用户名 -> 存储账户名称
密码 -> 访问密钥

【Azure 文件同步】
* 基于 Azure 文件共享
* 在本地缓存文件的副本
* 多个分支机构在不同的位置搭建缓存服务器
配置步骤：创建文件同步服务->创建同步组->添加云端点（Azure文件共享）->安装并注册本地Windows Server->添加本地服务器为服务器终结点
